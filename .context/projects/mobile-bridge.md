# Project: Mobile Bridge — Drive Mobile Web Users to the App

**Squad**: Growth Acquisition  
**Status**: In delivery (Frontsite experiment active as of March 2026)  
**One-Pager**: Acuity One-Pager: Drive mobile web users to the app (March 2026)

## Problem

Mobile web has 320K landings/month but converts at only 6.90% TTS vs. 11.9% on desktop. The mobile app, despite minimal product investment, outperforms mobile web on TTS (8.25%) and on every core setup action.

Organic app discovery is already happening: trialers are finding and installing the app on their own during their trial period before subscribing — without any deliberate nudge from Acuity.

## Hypothesis (H4)

A deliberate bridge strategy can extend and accelerate organic behavior that's already happening. By prompting mobile web users to install the app at the right moment, we can route motivated prospects to a better-conversion surface.

## What This Experiment Tests

A Frontsite experiment testing whether top-of-funnel mobile web users respond to a deliberate prompt to install the app.

**Two critical signals being measured:**
1. **Appetite**: Do mobile web prospects at the top of the funnel have interest in the app proposition, or is the install ask premature at that stage?
2. **Downstream signal**: Do users who bridge from Frontsite to the app convert and activate at higher rates than those who continue on mobile web?

## What Each Outcome Means

| Outcome | Interpretation | Next Investment |
|---------|---------------|-----------------|
| Strong install intent | Bridge strategy works; top-of-funnel is ready for the ask | Invest in post-install app onboarding experience |
| Weak install intent | Prospects need more value demonstration before app ask is compelling | Invest in mobile web onboarding/value demonstration layer first |

Either outcome advances the strategy — both are informative.

## Context: Why the App Outperforms Despite Being Mostly Webviews

Mobile app trialers complete setup actions at higher rates than mobile web trialers, despite:
- Nearly all setup functionality running in webviews (availability, appointment types, payments)
- Encountering the same or worse UX as mobile web in those webviews

This points to a genuine intent and engagement advantage in app users, not a superior product experience. The app's opportunity ceiling hasn't been tested with targeted native investments.
