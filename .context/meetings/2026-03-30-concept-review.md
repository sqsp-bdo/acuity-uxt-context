## Key Decision: Concept 2 Selected

**Concept 1 (Upfront Generation)** — Rejected  
Highly opinionated flow that assumed industry and pre-generated common appointment types, prices, durations. User selects/edits from pre-generated options.
- Upside: Speed, tangible result quickly
- Downside: Editing feels like "correcting" rather than "building." If output doesn't match expectations, experience breaks.
- Also rejected: free-text AI generation (insufficient knowledge of user interaction edge cases for first test)

**Concept 2 (Guided Step-Based Creation)** — Selected  
Less opinionated upfront. Breaks setup into smaller guided steps with smart defaults. User provides input; system generates content in context at each step (e.g., user types "Zumba class" → system provides an opinionated description the user can rewrite).

**Why Concept 2 won**: Team preferred user-provides-input direction over system-generates-potentially-unused-bundle. Andrej: "if generation is used, users need the ability to regenerate options."
